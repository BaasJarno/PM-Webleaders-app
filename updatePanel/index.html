Updates en diagnose


    

      Waarom faalt updaten soms?
      
    
        
  • Alleen de geïnstalleerde app (niet npm start) kan echt op GitHub controleren.
    • 
        
  • De GitHub-repository (of in elk geval de releases) moeten openbaar leesbaar zijn zonder inlog, tenzij je zelf auth regelt.
    • 
        
  • Op de nieuwste release moeten o.a. latest.yml (Windows) of latest-mac.yml (Mac) als bestand bij de release staan, naast de installer/zip.
    • 
        
  • Een release die alleen als pre-release is gemarkeerd, wordt soms overgeslagen; gebruik geen pre-release tenzij je weet dat je die nodig hebt.
    • 
        
  • Op je pc moet de geïnstalleerde versie lager zijn dan de release (semver); anders: “geen update”.
    • 
        
  • Netwerk, proxy of firewall kunnen GitHub blokkeren — de fout in het rood zegt meestal wat er misgaat (code, URL of netwerk).
    •
